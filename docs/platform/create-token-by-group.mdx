# 如何新建指定分组的令牌

这一页解决的是一个非常具体的问题：

**我已经知道自己要用哪个分组了，接下来怎么把它真正做成一个可用令牌。**

如果你还没决定分组，先别急着创建，优先看：

- [分组详细表格](/docs/platform/group-table)
- [分组的特殊性及价格差异](/docs/platform/group-price-differences)

## 创建前先准备 3 件事

- 你已经知道这次令牌是给谁用的
- 你已经知道应该落在哪个分组
- 你知道这个令牌会接哪个客户端或服务

不要在“用途不明确”的情况下直接点新建。  
最容易制造混乱的，就是先建一堆名字很像、分组也不明确的令牌。

## 推荐命名方式

令牌名称建议直接写成：

`用途 + 环境 + 客户端`

例如：

- `cursor-dev`
- `cherry-desktop`
- `backend-prod`
- `openwebui-team`

这样后面出问题时，你看日志和账单会省很多时间。

## 实际操作步骤

### 第 1 步：进入令牌管理页

先登录控制台，进入令牌管理页面。  
如果你当前账号没有新建权限，先不要继续往下配，先确认权限。

### 第 2 步：点击新建令牌

打开新建令牌弹窗后，先不要急着填所有高级选项，按这个顺序最稳：

1. 名称
2. 分组
3. 额度
4. 过期时间
5. 模型限制
6. 其他附加限制

### 第 3 步：明确选择分组

这是最关键的一步。  
一定要主动确认当前选中的就是你要的分组，而不是沿用上一次的默认值。

常见做法：

- Cursor 用编码助手或高稳定聊天分组
- Cherry Studio / Lobe Chat / Open WebUI 用通用聊天分组
- 后端生产服务用生产分组
- 离线批处理用低成本分组

### 第 4 步：设置额度

第一轮建议别配得太极端。

推荐思路：

- 本地测试：小额度
- 日常客户端：中等额度
- 生产服务：独立额度并单独观察

如果你不确定是否需要无限额度，先不要开。

### 第 5 步：设置过期时间

如果这个令牌只是临时测试，用短周期最稳。  
如果是长期客户端使用，再考虑更长的有效期。

### 第 6 步：按需要限制模型

如果你已经明确知道这个令牌只给某个客户端用，就只开放那个客户端会用到的模型。

例如：

- Cursor 只开放 1 到 3 个编码模型
- Cherry Studio 只开放几个聊天模型
- Open WebUI 先只开放聊天模型，再慢慢扩

### 第 7 步：创建后立刻复制并保存

创建完成后，立刻把令牌复制到你当前准备接入的地方，不要等后面再找。

推荐存放方式：

- 本地环境变量
- 密钥管理工具
- 服务端环境变量

不推荐直接丢进聊天记录、飞书群、临时文档或代码仓库。

## 创建后不要直接结束，先做这 2 个验证

### 验证 1：看模型列表

```bash
curl https://api.jingxing.uk/v1/models \
  -H "Authorization: Bearer <YOUR_TOKEN>"
```

你要看的不是“有没有返回”，而是：

- 返回的模型是否符合这个分组的预期
- 有没有出现不该开放的模型
- 有没有缺少你本来就要用的模型

### 验证 2：发第一条最小请求

```bash
curl https://api.jingxing.uk/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

这一步是为了确认：

- token 本身可用
- 分组和模型权限匹配
- 不是只停留在“创建成功但实际不能用”

## 最常见的 5 个错误

### 1. 选错分组

这是最常见也最隐蔽的问题。  
很多人以为自己在测模型，实际上是在测错误分组。

### 2. 复制了旧令牌

尤其是桌面客户端里最容易发生：你以为换了新 token，实际上客户端还在用旧值。

### 3. 模型限制太窄

新令牌建完以后 `GET /models` 什么都不对，大概率先看这里。

### 4. 额度给得太低

导致刚一验证就失败，然后误以为是分组问题。

### 5. 客户端和用途不匹配

比如给 Cursor 用了只适合低成本测试的分组，然后以为平台不稳定。

## 推荐动作

如果你现在就准备开始建，按这个顺序最稳：

1. 先确定用途
2. 再确定分组
3. 新建令牌
4. 跑 `GET /models`
5. 发最小请求
6. 最后再接入客户端
