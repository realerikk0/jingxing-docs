# API 概览

井陉 API 以 OpenAI 兼容接口为主，按能力划分为聊天、Responses、Embeddings、图片、视频和模型列表查询。

除特殊说明外，请求地址均以以下基础地址为前缀：

```txt
https://api.jingxing.uk
```

## 聊天

聊天接口用于多轮对话、文本生成、代码生成、信息抽取、内容改写、结构化回答等场景。大多数客户端和 OpenAI 兼容 SDK 会优先使用该接口。

对应路径：

- `POST /v1/chat/completions`

常见用途：

- 普通问答
- 多轮上下文对话
- 代码解释与生成
- 文档总结
- JSON 或结构化内容生成

## Responses

Responses 接口用于 OpenAI Responses 风格的模型调用，适合需要更统一响应结构、工具调用、推理流程或更现代 OpenAI 接口形态的场景。

对应路径：

- `POST /v1/responses`
- `POST /v1/responses/compact`

常见用途：

- 新版 OpenAI 风格应用接入
- 多轮任务处理
- 工具调用与复杂推理流程
- 长对话压缩与上下文整理

## Embeddings

<Admonition type="warning" title="暂未开放">
  Embeddings 能力目前暂未开放。页面保留接口路径用于后续接入说明，实际可用状态以控制台和当前分组权限为准。
</Admonition>

Embeddings 接口用于将文本转换为向量，常见于搜索、推荐、相似度匹配、知识库和 RAG 场景。

对应路径：

- `POST /v1/embeddings`
- `POST /v1/engines/{model}/embeddings`

常见用途：

- 文档向量化
- 语义搜索
- 相似问题匹配
- 知识库检索
- RAG 检索增强生成

## 图片

图片接口用于生成图片或基于已有图片进行编辑。适合产品配图、创意素材、视觉草图、封面图和图像修改等场景。

对应路径：

- `POST /v1/images/generations`
- `POST /v1/images/edits`

常见用途：

- 文生图
- 图片局部编辑
- 风格化生成
- 视觉素材批量生成
- 营销图、封面图、示意图制作

## 视频

<Admonition type="warning" title="暂未开放">
  视频能力目前暂未开放。页面保留接口路径用于后续接入说明，实际可用状态以控制台和当前分组权限为准。
</Admonition>

视频接口用于提交视频生成任务，并通过任务 ID 查询生成状态或获取结果内容。

对应路径：

- `POST /v1/videos`
- `GET /v1/videos/{task_id}`
- `GET /v1/videos/{task_id}/content`
- `POST /kling/v1/videos/text2video`
- `GET /kling/v1/videos/text2video/{task_id}`
- `POST /kling/v1/videos/image2video`
- `GET /kling/v1/videos/image2video/{task_id}`

常见用途：

- 文生视频
- 图生视频
- 视频任务状态查询
- 生成结果下载

## 模型列表查询

模型列表查询用于查看当前令牌、分组和权限下可见的模型。首次接入、客户端配置、模型不可用排查时，建议先调用该接口。

对应路径：

- `GET /v1/models`
- `GET /v1beta/models`

常见用途：

- 验证 API Key 是否有效
- 查看当前令牌可用模型
- 确认可用分组和模型范围
- 排查模型名称填写错误
- 在客户端中同步模型列表
