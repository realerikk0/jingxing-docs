# API 概览

本文用于说明井陉当前 API 的主要类别、各自适用场景，以及文档阅读入口与 OpenAPI 规范之间的关系。

## API 类别

井陉当前可分为以下两类 API：

### 中转 API

中转 API 主要用于模型能力接入，覆盖以下典型场景：

- 聊天
- Responses
- Embeddings
- 图片
- 视频
- 模型列表查询

当前仓库中同步的规范文件为：

- `static/openapi/relay.json`

### 后台管理 API

后台管理 API 主要用于平台侧管理、控制台联动和自动化管理脚本，关注内容包括：

- 用户
- 分组
- 模型
- 渠道
- 配置
- 登录、注册与状态信息

当前仓库中同步的规范文件为：

- `static/openapi/api.json`

## 阅读建议

### 面向普通使用者

如目标是接入模型、验证接口或接入客户端，建议优先阅读中转 API 相关文档。

### 面向平台管理或系统联动

如目标是进行控制台二次开发、平台联动或自动化管理，建议进一步阅读后台管理 API 相关内容。

## 中转 API 中的高频路径

根据当前仓库同步的 `relay.json`，建议优先关注以下路径：

- `GET /v1/models`
- `POST /v1/chat/completions`
- `POST /v1/responses`
- `POST /v1/embeddings`
- `POST /v1/images/generations`
- `POST /v1/images/edits`
- `POST /v1/videos`
- `GET /v1/videos/{task_id}`

在首次验证阶段，建议始终先从 `GET /v1/models` 开始。

## 后台管理 API 的典型适用场景

以下场景通常需要阅读后台管理 API：

- 控制台相关二次开发
- 平台联动
- 自动化管理脚本
- 用户、模型、渠道或配置管理

如当前目标仅为调用模型能力，可暂不阅读该部分。

## 文档与 OpenAPI 的关系

本文档页面主要用于说明：

- 当前应阅读哪一类接口
- 接入顺序应如何安排
- 各类能力的定位与范围

OpenAPI 规范主要用于说明：

- 具体字段定义
- 请求与响应格式
- 参数结构与接口明细

因此，文档页负责导航与说明，OpenAPI 规范负责接口细节。
