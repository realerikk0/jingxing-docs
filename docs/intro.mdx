# 井陉doc

井陉是一个统一制式的 AI 网关与工作台，面向 API 调用、模型分组、令牌管理、额度计费与开发工具集成。你可以把它理解成一层稳定、可控、可审计的模型接入层：对外提供一致的调用方式，对内管理不同供应商、不同模型和不同分组策略。

## 适合谁

- 希望通过统一接口接入多个模型系列的开发者
- 需要按团队、项目或环境隔离令牌与额度的管理员
- 想把 Cursor、Cherry Studio、Lobe Chat、Claude Code 等客户端统一接到同一套网关的人

## 井陉能解决什么问题

- 用统一的 `Base URL` 和鉴权方式接入多个模型
- 用分组表达价格、稳定性、可用模型范围和团队策略
- 把 API 令牌、模型权限、额度与调用审计放到同一处管理
- 用一致的 OpenAI 兼容方式降低客户端迁移成本

## 快速事实

- 控制台地址：`https://jingxing.uk`
- API 基础地址：`https://api.jingxing.uk/v1`
- 默认接入方式：OpenAI 兼容接口
- 令牌形式：Bearer Token
- 能力范围：聊天、Embedding、图片、视频、音频/Realtime，以及控制台管理接口
