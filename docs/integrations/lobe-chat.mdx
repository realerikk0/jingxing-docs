# Lobe Chat

Lobe Chat 是一款网页端 AI 聊天应用，也支持自部署为团队或个人的聊天前端。它可以通过 OpenAI provider 接入 OpenAI 兼容接口。接入井陉后，用户可以在 Lobe Chat 中使用井陉令牌调用可用模型，并在井陉控制台查看调用记录与额度消耗。

## 打开 Lobe Chat

如果只是快速体验，可以直接使用 LobeHub 官方应用：

- [Lobe Chat 官方应用](https://app.lobehub.com/)
- [LobeHub 下载页](https://lobehub.com/downloads/)

如需要长期作为团队聊天入口，建议使用自部署版本。自部署版本可以通过环境变量固定 API endpoint、令牌和模型列表，更适合稳定管理和统一维护。

## 适用场景

- 希望在浏览器中快速使用井陉模型。
- 希望搭建一个统一的 Web 聊天前端。
- 希望把聊天记录、模型选择和常用助手集中在 Lobe Chat 中管理。
- 希望通过井陉令牌统一查看调用、额度和消耗。

## 前置条件

开始前，请先准备以下信息：

| 项目 | 说明 |
| --- | --- |
| Lobe Chat | 可使用官方应用，或已有自部署环境 |
| API endpoint | `https://api.jingxing.uk/v1` |
| 井陉令牌 | 在“令牌管理”中创建的 `sk-` 开头令牌 |
| 模型名称 | 当前令牌可用的模型名称，以控制台显示为准 |

建议为 Lobe Chat 单独创建令牌，不要复用生产服务、团队共享或其他客户端正在使用的高权限令牌。

## 接入方式总览

| 方式 | 适合场景 | 配置方式 |
| --- | --- | --- |
| 控制台一键打开 | 快速体验、个人使用 | 从“令牌管理”的聊天入口打开 |
| 自部署配置 | 长期使用、团队入口、固定配置 | 设置 `OPENAI_API_KEY` 和 `OPENAI_PROXY_URL` |

## 方式一：从控制台一键打开

如果你只需要快速打开 Lobe Chat，可以从井陉控制台进入：

```txt
令牌管理 -> 选择一个令牌 -> 聊天下拉菜单 -> Lobe Chat 官方示例
```

系统会根据当前令牌生成对应的打开链接，并尝试把配置带入 Lobe Chat。建议使用新窗口打开 `app.lobehub.com`，如果页面要求登录，请先完成 Lobe Cloud 登录。

适合这种方式的情况：

- 只需要快速验证聊天是否可用。
- 不想先部署自己的 Lobe Chat。
- 接受官方应用的登录、配置和页面行为。

如果打开后没有出现目标模型，或配置没有自动生效，请改用下方自部署方式进行固定配置。

## 方式二：自部署接入

自部署适合长期使用。它可以把井陉 endpoint、令牌和模型列表写入部署环境，避免每个用户重复手动配置。

### 1. 创建 Lobe Chat 专用令牌

进入井陉控制台：

```txt
令牌管理 -> 新建令牌
```

建议命名为 `lobe-chat`、`lobe-web`、`lobe-team` 等便于识别的名称。后续查看 Lobe Chat 消耗、停用令牌或排查请求时，可以直接定位到对应令牌。

### 2. 设置 OpenAI provider 环境变量

在 Lobe Chat 的自部署环境中设置：

```bash title=".env"
OPENAI_API_KEY="<你的井陉令牌>"
OPENAI_PROXY_URL="https://api.jingxing.uk/v1"
```

其中：

| 变量 | 说明 |
| --- | --- |
| `OPENAI_API_KEY` | 井陉令牌，例如 `sk-...` |
| `OPENAI_PROXY_URL` | 井陉 API endpoint，需要包含 `/v1` |

请勿把真实令牌写入公开文档、截图、代码仓库或聊天记录中。

### 3. 配置模型列表

如果 Lobe Chat 没有自动显示目标模型，可以使用 `OPENAI_MODEL_LIST` 显式控制模型列表：

```bash title=".env"
OPENAI_MODEL_LIST="-all,+<模型名称>"
```

示例：

```bash title=".env"
OPENAI_MODEL_LIST="-all,+gpt-4.1"
```

模型名称必须与井陉控制台显示一致。首次验证建议只添加一个模型，确认链路正常后再逐步扩展。

### 4. 重启 Lobe Chat

修改环境变量后，需要重新启动或重新部署 Lobe Chat。完成后进入 Lobe Chat 页面，选择 OpenAI provider 下的目标模型，并发送一条测试消息：

```txt
请用一句话介绍你自己。
```

如果 Lobe Chat 能正常返回，并且井陉控制台中出现调用记录，说明接入已经完成。

## 界面示意

以下界面示意来自 LobeHub 官方文档的模型服务商环境变量章节，用于说明接入兼容网关时需要关注的核心变量。

![Lobe Chat 模型服务商环境变量示意](/img/integrations/lobe-model-provider-doc.png)

与井陉接入直接相关的字段包括：

- `OPENAI_API_KEY`：填写井陉令牌。
- `OPENAI_PROXY_URL`：填写 `https://api.jingxing.uk/v1`。
- `OPENAI_MODEL_LIST`：用于控制 Lobe Chat 中可见的模型列表。

## 验证方法

推荐按以下顺序验证：

1. 在井陉控制台确认令牌处于启用状态。
2. 确认 `OPENAI_API_KEY` 已设置为井陉令牌。
3. 确认 `OPENAI_PROXY_URL` 已设置为 `https://api.jingxing.uk/v1`。
4. 如需固定模型，确认 `OPENAI_MODEL_LIST` 中的模型名称正确。
5. 重启或重新部署 Lobe Chat。
6. 在 Lobe Chat 中选择目标模型并发送测试消息。
7. 回到井陉控制台查看调用记录和额度消耗。

## 注意事项

- 官方托管版更适合快速体验；长期团队使用建议自部署。
- 自部署环境变量修改后通常需要重启或重新部署才会生效。
- `OPENAI_PROXY_URL` 需要包含 `/v1`，否则可能出现空响应或请求路径错误。
- 模型名称必须与井陉控制台显示一致，不要自行改写大小写、前缀或后缀。
- 不建议在同一个部署中混用多个不相关用途的令牌。

## 常见问题

### 官方应用可以直接使用吗

可以用于快速体验。建议从井陉控制台的“聊天下拉菜单”打开 `Lobe Chat 官方示例`，系统会尝试带入当前令牌对应的配置。若需要长期稳定使用，建议自部署。

### 自部署时 Base URL 应该填写什么

在 Lobe Chat 的环境变量中填写：

```txt
OPENAI_PROXY_URL=https://api.jingxing.uk/v1
```

LobeHub 官方文档说明，`OPENAI_PROXY_URL` 用于覆盖默认 OpenAI API 请求基础 URL。井陉的 OpenAI 兼容接口位于 `/v1`，因此这里需要包含 `/v1`。

### 模型列表中没有目标模型

请检查：

- 当前令牌是否有目标模型权限。
- `OPENAI_MODEL_LIST` 是否隐藏了目标模型。
- 模型名称是否与井陉控制台显示一致。
- 修改环境变量后是否已经重启或重新部署。

### 页面可以打开，但聊天失败

常见原因包括：

- `OPENAI_API_KEY` 不是有效的井陉令牌。
- `OPENAI_PROXY_URL` 没有包含 `/v1`。
- 当前模型不可用或令牌无权限。
- 部署环境没有读取到最新环境变量。

建议先用一个已确认可调用的模型完成最小验证，再继续排查目标模型。

### 如何确认请求是否经过井陉

最直接的方式是在井陉控制台查看调用记录。若 Lobe Chat 返回成功，但控制台没有记录，通常说明当前页面没有使用井陉 endpoint，或正在使用其他 provider 配置。
