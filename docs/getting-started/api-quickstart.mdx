# API 快速开始

井陉优先提供 OpenAI 兼容接口，便于在尽量少改动现有代码和客户端配置的前提下完成接入。本文用于完成首次连通性验证，并建立最基本的调用链路。

## 接入信息

### Base URL

`https://api.jingxing.uk/v1`

### 鉴权方式

所有公共接口均使用 Bearer Token：

```http
Authorization: Bearer <YOUR_TOKEN>
```

## 第一步：获取模型列表

建议首先执行模型列表查询，以确认令牌、网络和接口地址均已生效。

```bash
curl https://api.jingxing.uk/v1/models \
  -H "Authorization: Bearer <YOUR_TOKEN>"
```

该请求主要用于确认以下内容：

- 令牌是否有效
- 当前令牌可见的模型范围
- 当前环境是否可以访问井陉 API

## 第二步：发送最小聊天请求

完成模型列表验证后，建议使用最小聊天请求确认主链路已建立。

```bash
curl https://api.jingxing.uk/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "gpt-5.4",
    "messages": [
      {"role": "user", "content": "请简要介绍井陉。"}
    ]
  }'
```

## 请求要点

### 模型名称

`model` 字段必须填写当前分组和当前令牌实际可用的模型名称。模型是否可见通常取决于以下因素：

- 当前分组策略
- 当前令牌的模型限制
- 后台是否启用了对应上游渠道

### OpenAI 兼容接入范围

对于大多数现有集成，迁移至井陉时通常只需调整以下三项：

- Base URL
- API Key
- model

### 请求头

首次接入时，最常见的问题集中在请求头配置不完整。建议确认：

- 已携带 `Authorization: Bearer <YOUR_TOKEN>`
- 已携带 `Content-Type: application/json`

## 建议接入顺序

建议采用以下顺序完成接入：

1. 先执行 `GET /v1/models`
2. 再执行 `POST /v1/chat/completions`
3. 再接入 SDK、服务端或客户端
4. 最后扩展流式输出、工具调用、图片或其他能力

## 常见问题

### 401 未认证

通常与令牌失效、请求头缺失或使用了错误的令牌类型有关。

### 400 参数错误

建议优先检查：

- `model` 是否正确
- `messages` 是否为合法数组
- JSON 结构是否完整

### 404 或无法连通

常见原因是接口地址未包含 `/v1` 前缀。应使用：

- `https://api.jingxing.uk/v1`

而非：

- `https://api.jingxing.uk`
