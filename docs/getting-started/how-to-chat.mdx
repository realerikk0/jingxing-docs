# 如何聊天

井陉控制台提供“聊天”入口，用于把当前令牌一键带入常用客户端或网页应用。用户不需要手动复制后台密钥，也不需要自己拼接配置链接。

## 入口在哪里

进入控制台后，打开：

```txt
令牌管理 -> 选择一个令牌 -> 聊天下拉菜单
```

在下拉菜单中选择目标应用后，系统会根据当前令牌生成对应的打开链接。不同应用的打开方式不同，有的会拉起本机软件，有的会打开网页，有的属于特殊集成入口。

## 聊天方式总览

| 应用 | 类型 | 推荐打开方式 | 是否需要安装 |
| --- | --- | --- | --- |
| Cherry Studio | 桌面聊天客户端 | 打开应用按钮 | 需要 |
| AionUI | 桌面 AI 工作台 | 打开应用按钮 | 需要 |
| 流畅阅读 | 浏览器双语翻译插件 | 特殊集成入口 | 需要浏览器插件 |
| CC Switch | AI 编程工具配置管理器 | 特殊集成入口 | 需要 |
| Lobe Chat 官方示例 | 网页聊天应用 | 打开应用按钮（新窗口） | 不需要本地安装，可能需要登录 |
| AI as Workspace | 网页 / PWA AI 工作台 | iframe | 不需要本地安装 |
| AMA 问天 / BotGem | 移动与桌面聊天客户端 | 打开应用按钮 | 需要 |
| OpenCat | Mac / iOS / iPad 聊天客户端 | 打开应用按钮 | 需要 |

## Cherry Studio

Cherry Studio 是一款桌面 AI 聊天客户端，支持自定义 OpenAI 兼容服务商。适合在本地桌面环境中使用多个模型，并把井陉作为统一 API 入口。

下载地址：

- [Cherry Studio 官方下载](https://docs.cherry-ai.com/en-us/cherrystudio/download)

推荐使用方式：

1. 先安装并打开 Cherry Studio。
2. 在井陉控制台进入“令牌管理”。
3. 选择要使用的令牌。
4. 在“聊天”下拉菜单中点击 `Cherry Studio`。
5. 浏览器弹出外部应用确认时，选择允许打开。
6. Cherry Studio 拉起后，检查服务商配置是否已导入。

注意事项：

- 如果没有安装 Cherry Studio，点击后通常只会停留在浏览器页面。
- 浏览器通常要求用户主动点击后才允许拉起本机应用，因此建议从按钮或下拉菜单点击打开。

## AionUI

AionUI 是面向 AI 编程与工作流的桌面工作台，可连接 Gemini CLI、Claude Code、Codex、Qwen Code 等工具，也支持本地会话和多平台使用。

下载地址：

- [AionUI 官方网站](https://aionui.site/)
- [AionUI GitHub Releases](https://github.com/iOfficeAI/AionUi/releases)

推荐使用方式：

1. 先安装并打开 AionUI。
2. 在井陉控制台进入“令牌管理”。
3. 选择要使用的令牌。
4. 在“聊天”下拉菜单中点击 `AionUI`。
5. 浏览器提示打开外部应用时，选择允许。
6. AionUI 打开后，确认 provider 配置已导入。

适合场景：

- 希望在桌面工作台中管理 AI 会话。
- 需要把井陉配置接入本地 AI 工具链。
- 希望减少手动填写 Base URL 和 API Key。

## 流畅阅读

流畅阅读是一款浏览器双语翻译插件，适合网页阅读、划词翻译和双语对照阅读场景。它不是普通网页聊天应用，而是浏览器扩展类集成入口。

下载地址：

- [流畅阅读官方网站](https://fluent.thinkstu.com/)

推荐使用方式：

1. 先按浏览器类型安装流畅阅读扩展。
2. 在井陉控制台进入“令牌管理”。
3. 选择要用于翻译或阅读场景的令牌。
4. 在“聊天”下拉菜单中点击 `流畅阅读`。
5. 按页面提示完成集成。

注意事项：

- `fluentread` 是特殊集成入口，不是普通 URL。
- 不建议把它复制到浏览器地址栏中当作网页访问。
- 如点击后没有反应，请先确认扩展已安装并启用。

## CC Switch

CC Switch 是一款桌面工具，用于统一管理 Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw 等 AI 编程工具的 provider 配置。它更适合开发者把井陉配置导入到本地编程工具链。

下载地址：

- [CC Switch 官方网站](https://ccswitch.ai/)
- [CC Switch GitHub Releases](https://github.com/farion1231/cc-switch/releases)

推荐使用方式：

1. 先安装并打开 CC Switch。
2. 在井陉控制台进入“令牌管理”。
3. 选择要用于编程工具的令牌。
4. 在“聊天”下拉菜单中点击 `CC Switch`。
5. 按 CC Switch 的提示选择目标工具和配置名称。
6. 导入后，在 CC Switch 中切换到对应 provider。

注意事项：

- `ccswitch` 是特殊集成入口，不是普通 URL。
- 它主要用于配置 AI 编程工具，不是普通聊天网页。
- 如果系统没有安装 CC Switch，对应入口无法完成导入。

## Lobe Chat 官方示例

Lobe Chat 是网页端 AI 聊天应用。井陉中的 Lobe Chat 入口会把配置带到 `app.lobehub.com`，适合用户在浏览器中快速体验聊天。

打开地址：

- [Lobe Chat 官方示例](https://app.lobehub.com/)
- [LobeHub 下载页](https://lobehub.com/downloads/)

推荐使用方式：

1. 在井陉控制台进入“令牌管理”。
2. 选择要使用的令牌。
3. 在“聊天”下拉菜单中点击 `Lobe Chat 官方示例`。
4. 建议使用新窗口打开。
5. 如页面要求登录，请先完成 Lobe Cloud 登录。
6. 打开后检查模型和服务商配置是否生效。

注意事项：

- Lobe Chat 是网页应用，不需要本机协议拉起。
- 官方示例站点可能要求登录 Lobe Cloud。
- 如浏览器阻止弹窗，请允许当前站点打开新窗口。

## AI as Workspace

AI as Workspace 是一个轻量、可扩展的 AI 客户端，支持 Web、PWA 和本地客户端形态。井陉中的该入口适合作为网页应用打开，也可以作为网页嵌入使用。

打开地址：

- [AI as Workspace](https://aiaw.app/)
- [AI as Workspace 文档](https://docs.aiaw.app/)

推荐使用方式：

1. 在井陉控制台进入“令牌管理”。
2. 选择要使用的令牌。
3. 在“聊天”下拉菜单中点击 `AI as Workspace`。
4. 系统会打开或嵌入 AI as Workspace 页面。
5. 页面加载后确认 provider 配置是否已生效。

适合场景：

- 希望直接在网页中聊天。
- 希望把聊天界面嵌入控制台页面。
- 不想先安装桌面客户端。

## AMA 问天 / BotGem

AMA 问天现已更名为 BotGem，是一款移动与桌面 AI 聊天客户端，适合在手机、平板或桌面端快速进行对话。

下载地址：

- [BotGem 官方网站](https://botgem.com/)
- [BotGem App Store 页面](https://apps.apple.com/hk/app/botgem/id6446135619)

推荐使用方式：

1. 先安装 BotGem。
2. 在井陉控制台进入“令牌管理”。
3. 选择要使用的令牌。
4. 在“聊天”下拉菜单中点击 `AMA 问天`。
5. 浏览器提示打开外部应用时，选择允许。
6. BotGem 打开后，确认服务器地址和令牌配置已写入。

注意事项：

- `ama://set-api-key?...` 属于 App 协议链接。
- 必须先安装客户端，浏览器才能拉起应用。
- 如移动端浏览器拦截外部 App 打开，请换用系统浏览器重试。

## OpenCat

OpenCat 是 Mac、iOS、iPad 上的 AI 聊天客户端，支持自定义 API Key 和团队加入链接。井陉中的 OpenCat 入口会通过 App 协议导入服务器地址和令牌。

下载地址：

- [OpenCat 官方网站](https://opencat.app/)

推荐使用方式：

1. 先安装 OpenCat。
2. 在井陉控制台进入“令牌管理”。
3. 选择要使用的令牌。
4. 在“聊天”下拉菜单中点击 `OpenCat`。
5. 浏览器提示打开 OpenCat 时，选择允许。
6. OpenCat 打开后，确认已加入对应配置。

注意事项：

- `opencat://team/join?...` 属于 App 协议链接。
- 未安装 OpenCat 时，浏览器通常无法打开该链接。
- 如系统提示是否允许打开外部 App，需要手动确认。

## 常见问题

### 点击后没有反应

常见原因包括：

- 本机尚未安装对应客户端。
- 浏览器阻止了外部 App 协议。
- 不是从按钮主动点击打开。
- 当前应用不支持所在设备或系统。

建议回到“令牌管理”的聊天下拉菜单重新点击，不建议手动复制链接到地址栏。

### 为什么不能直接复制链接给别人

聊天打开链接中可能包含当前令牌生成的配置。该配置仅用于当前用户自己的客户端接入，不应复制给其他人，也不应公开到截图、群聊、工单或文档中。

### 为什么网页应用和桌面应用打开方式不同

网页应用通常可以直接在浏览器中打开；桌面或移动客户端通常依赖 `cherrystudio://`、`aionui://`、`ama://`、`opencat://` 这类 App 协议。App 协议必须由已安装的客户端处理。

## 安全建议

- 不要在公开页面展示真实 token。
- 不要把聊天配置链接发给无关人员。
- 不要把同一个令牌同时用于多个不相关客户端。
- 如怀疑令牌泄露，应立即在“令牌管理”中停用或重新生成。
