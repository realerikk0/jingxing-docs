# 核心概念

本文用于说明井陉文档中最常出现的几个核心概念，并明确它们之间的关系。

## 模型

模型表示最终被调用的能力实体，例如：

- `gpt-5.4`
- `claude-sonnet-4-6`
- `gemini-3-flash-preview`

模型决定“调用的是哪一种能力”。

## 分组

分组不是简单的名称标签，而是一组实际生效的策略集合，通常决定以下内容：

- 哪些模型可见
- 价格如何计算
- 当前链路更偏向成本还是稳定性
- 更适合哪类客户端或任务

分组决定“在什么规则下使用模型”。

## 令牌

令牌是客户端或程序调用 API 时使用的凭证。除鉴权外，令牌通常还承担以下策略边界：

- 所属分组
- 额度控制
- 模型限制
- 过期时间
- 来源限制

令牌决定“谁可以在什么边界下调用模型”。

## 额度

额度是平台内部统一使用的记账单位，用于汇总不同模型、不同能力和不同供应商的消耗。用户通常需要重点关注：

- 当前剩余额度
- 额度变化速度
- 具体由哪类请求产生消耗

## 价格

价格用于定义额度与实际成本之间的换算关系。价格通常受到以下因素共同影响：

- 模型本身
- 分组策略
- 能力类型

因此，同为聊天模型，也可能由于分组或能力差异而呈现不同成本。

## 概念之间的关系

在实际操作中，通常遵循以下顺序：

1. 先确定分组
2. 再创建令牌
3. 再使用令牌调用模型
4. 最后在额度与账单中观察消耗结果

## 常见混淆点

### 模型与分组混淆

模型表示能力本体，分组表示应用该能力时的策略与边界，两者并不等同。

### 控制台登录与令牌混淆

控制台登录用于网页身份认证，令牌用于 API 调用，两者用途不同。

### 价格与额度混淆

价格决定换算规则，额度是换算后的统一结果，两者不应视为同一概念。
