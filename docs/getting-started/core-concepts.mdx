# 核心概念

这一页不讲抽象定义，而是帮你把文档里最常出现的几个词一次看懂。  
如果你第一次读到“分组、令牌、额度、模型”这些词容易混，这页就是给你准备的。

## 你会反复遇到的 5 个词

### 模型

模型就是最终被调用的能力本体。  
例如：

- `gpt-5.4`
- `claude-sonnet-4-6`
- `gemini-3-flash-preview`

如果你把井陉理解成一个“模型入口层”，那模型就是你最终真正想用的那台“机器”。

### 分组

分组不是名字标签，而是一套策略。  
它通常一起决定：

- 哪些模型可见
- 价格怎么计算
- 更偏低成本还是更偏稳定
- 更适合哪类客户端或任务

一句话理解：

- 模型决定“用谁”
- 分组决定“怎么用、给谁用、按什么规则用”

### 令牌

令牌就是客户端和程序真正调用 API 时用的钥匙。

它可以绑定：

- 分组
- 额度
- 模型限制
- 过期时间
- 来源限制

所以令牌不是单纯的字符串，它本质上也是一层策略边界。

### 额度

额度是平台内部统一的记账单位。  
它的作用是把不同模型、不同能力、不同供应商的消耗折算到同一套账本里。

用户真正最常关心的是：

- 还剩多少
- 为什么掉得这么快
- 到底是哪一类请求在消耗

### 价格

价格决定额度与人民币或美元之间的换算关系，需要跟控制台和充值页保持一致。

更准确地说，价格通常受 3 层影响：

- 模型本身
- 分组策略
- 能力类型

所以“同样是聊天模型”不代表成本就一定一样。

## 这些词在真实操作里怎么串起来

用户真正的操作路径通常是：

1. 先选分组
2. 再建令牌
3. 再用令牌调用某个模型
4. 最后在额度和账单里看消耗

所以在井陉里，不太建议反过来理解成：

“我先拿个 token，其他以后再说。”

## 最容易混淆的地方

### 误区 1：模型和分组是一回事

不是。  
模型是能力，分组是规则。

### 误区 2：控制台登录和令牌是一回事

也不是。  
控制台登录是网页身份，令牌是 API 凭证。

### 误区 3：价格和额度是一回事

也不是。  
价格决定怎么换算，额度是换算后的统一结果。
