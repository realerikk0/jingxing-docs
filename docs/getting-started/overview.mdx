# 文档总览

本文用于说明井陉文档的适用对象与推荐阅读路径，帮助在不同任务场景下快速定位对应内容。

## 推荐阅读路径

### API 首次接入

如目标是完成首次 API 调用，建议按以下顺序阅读：

1. [快速开始](/docs/getting-started/quickstart)
2. [API 快速开始](/docs/getting-started/api-quickstart)
3. [鉴权方式](/docs/api-reference/authentication)
4. [令牌管理](/docs/platform/token-management)

### 第三方客户端接入

如目标是接入桌面客户端或 Web 前端，建议直接进入对应客户端页面：

- [Cursor](/docs/integrations/cursor)
- [Cherry Studio](/docs/integrations/cherry-studio)
- [Lobe Chat](/docs/integrations/lobe-chat)
- [Open WebUI](/docs/integrations/open-webui)
- [NextChat](/docs/integrations/nextchat)

### 错误排查

如目标是定位接口或客户端报错，建议优先阅读：

1. [HTTP 状态码总览](/docs/faq/http-status-codes)
2. [报错与排查 FAQ](/docs/faq/errors-and-debugging)
3. [错误与重试](/docs/platform/errors-and-retries)

## 适用对象

本文档适用于以下角色：

- 需要快速接入 API 的开发者
- 需要管理 API Key、令牌和额度的管理员
- 需要将第三方客户端统一接入井陉的使用者
- 需要查询价格、鉴权、错误与安全说明的运营或支持人员

## 阅读建议

建议按照当前任务目标选择阅读路径，而非顺序通读全部页面：

- 接 API：优先阅读基础与中转站基本介绍
- 接客户端：优先阅读开发工具
- 排查问题：优先阅读 FAQs
- 管理成本与权限：优先阅读分组、价格、额度与令牌相关页面
