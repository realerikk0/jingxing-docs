# 文档总览

如果你第一次打开井陉文档，最容易遇到的问题不是“内容太少”，而是“不知道我现在应该先看哪一页”。  
这页的作用就是把阅读顺序直接告诉你。

## 先说最短路线

### 如果你只是想先跑通

按这个顺序最省时间：

1. [快速开始](/docs/getting-started/quickstart)
2. [API 快速开始](/docs/getting-started/api-quickstart)
3. [分组详细表格](/docs/platform/group-table)
4. [如何新建指定分组的令牌](/docs/platform/create-token-by-group)

### 如果你主要是接客户端

直接跳到你要用的客户端页：

- [Cursor](/docs/integrations/cursor)
- [Cherry Studio](/docs/integrations/cherry-studio)
- [Lobe Chat](/docs/integrations/lobe-chat)
- [Open WebUI](/docs/integrations/open-webui)
- [NextChat](/docs/integrations/nextchat)

### 如果你现在是在排错

优先看：

1. [HTTP 状态码总览](/docs/faq/http-status-codes)
2. [报错与排查 FAQ](/docs/faq/errors-and-debugging)
3. [错误与重试](/docs/platform/errors-and-retries)

## 这套文档适合谁

- 想快速接 API 的开发者
- 想管理令牌、分组、额度的运营或管理员
- 想把常见客户端接到井陉的人
- 想查报错、价格、鉴权和安全问题的人

## 现在的导航该怎么理解

### 基础

适合第一次来的人。  
这里主要解决：

- 先跑通
- 先弄明白几个关键名词
- 先知道从哪里开始

### 中转站基本介绍

这是整套文档里最偏“运营手册”的部分。  
你会在这里看到：

- 分组怎么选
- 价格为什么不同
- 令牌怎么建
- 鉴权怎么排
- 错误怎么查

### 开发工具

如果你不是要自己写代码，而是要接：

- Cursor
- Claude Code
- Cherry Studio
- Lobe Chat
- Open WebUI
- NextChat

那这块最值得直接看。

### FAQs

这里是高频问题速查区。  
不讲大而全，优先讲“现在这个问题先做什么”。

## 推荐阅读方式

不要按“我想把所有页面都看完”的方式读。  
更好的方式是按任务来：

- 我要接 API：看基础 + 中转站基本介绍
- 我要接客户端：看开发工具
- 我报错了：看 FAQs
- 我想控制成本：看分组、价格、额度、令牌

- 想马上开始：前往[快速开始](/docs/getting-started/quickstart)
