# Kling 图生视频

井陉当前已提供 Kling 风格的图生视频接口，用于基于参考图像生成视频。本文用于说明最小请求、参考图输入方式、状态查询与典型问题。

## 可用路径

当前仓库已明确暴露以下路径：

- `POST /kling/v1/videos/image2video`
- `GET /kling/v1/videos/image2video/{task_id}`

## 最小请求

```bash
curl https://api.jingxing.uk/kling/v1/videos/image2video \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "kling-v1",
    "prompt": "人物缓慢转身并向前走一步，衣角轻微摆动，镜头稳定",
    "image": "https://example.com/reference-image.jpg",
    "duration": 5,
    "width": 1280,
    "height": 720
  }'
```

根据当前 OpenAPI 说明，`image` 字段支持：

- 图片 URL
- Base64 编码图片

首次验证建议优先使用稳定的公网图片地址。

## 状态查询

```bash
curl https://api.jingxing.uk/kling/v1/videos/image2video/<TASK_ID> \
  -H "Authorization: Bearer <YOUR_TOKEN>"
```

任务完成后，返回中通常可见：

- `url`
- `format`

## 适用任务类型

该接口更适合以下场景：

- 轻量动作驱动
- 环境动态增强
- 镜头缓动

在首次验证阶段，建议优先使用动作少、变量少的请求，以便快速确认主链路是否正常。

## 常见问题

### 参考图过于复杂

当参考图中主体过多或构图复杂时，结果稳定性通常会下降。

### 指令目标过于宽泛

图生视频更适合让现有画面产生运动，而不适合在首轮测试中完全重构一段新视频内容。

### 图片地址不稳定

过期链接、受限链接或响应缓慢的图片资源都会影响任务稳定性。

### 首次验证即使用长视频

建议先使用短时长建立主链路，再逐步扩展时长与复杂度。

### 混用图生视频与文生视频路径

提交和查询均应使用 `image2video` 路径，不应与 `text2video` 混用。

## 使用建议

建议按以下顺序验证图生视频链路：

1. 使用单主体参考图
2. 编写单一动作提示词
3. 使用较短时长
4. 获取 `task_id` 并查询状态
5. 成功后再扩展复杂镜头
