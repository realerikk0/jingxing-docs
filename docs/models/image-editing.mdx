# 编辑图片

这一页解决的是：

**我已经有一张参考图了，想让井陉按描述去修改它，应该怎么发请求。**

当前项目里已经明确暴露了 `POST /v1/images/edits`，OpenAPI 示例使用的是 `qwen-image-edit-plus`。  
如果你在当前分组里看不到这个模型，先不要继续改参数，先去确认分组和模型权限。

## 什么时候先看这页

- 你要做“给现有图片改风格 / 改背景 / 改主体”
- 你在测试图像编辑能力
- 你要验证参考图 URL 是否能被正常读取

## 最小请求

```bash
curl https://api.jingxing.uk/v1/images/edits \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "qwen-image-edit-plus",
    "input": {
      "messages": [
        {
          "role": "user",
          "content": [
            {
              "image": "https://example.com/source-image.jpg"
            },
            {
              "text": "保留主体构图，把背景改成雨夜街头霓虹灯氛围，整体更有电影感"
            }
          ]
        }
      ]
    },
    "parameters": {
      "n": 1,
      "prompt_extend": true,
      "watermark": false,
      "negative_prompt": ""
    }
  }'
```

## 你真正要检查的不是“有没有返回”，而是这 3 件事

- 平台能不能读到你的参考图
- 模型是不是按你的修改指令在动，而不是重画一张不相干的图
- 返回结果是不是稳定落在 `data[].url` 或 `data[].b64_json`

## 请求里最容易写错的地方

### `content` 里要同时放 `image` 和 `text`

这条接口不是单独传 `prompt`。  
你需要把参考图和修改说明都放进同一条消息里。

最稳的顺序就是：

1. 先放 `image`
2. 再放 `text`

### `image` 最好先用稳定的公网 URL

如果你一开始就用复杂的临时链接、过期签名或受限地址，排错会很痛苦。  
先用稳定 URL 把链路跑通，再换成你的正式资源。

### `n`

控制一次返回几张。  
第一次调试建议先用 `1`，不然你会把“请求问题”和“多结果选择问题”混在一起。

## 最适合先做的 3 类编辑

### 风格替换

例如把原图改成：

- 电影感
- 水彩风
- 3D 渲染
- 二次元海报

### 背景替换

例如：

- 室内改室外
- 白底改场景化背景
- 日景改夜景

### 轻度主体增强

例如：

- 保留人物姿态
- 提升服装细节
- 加强光影

## 最容易失败的 4 个原因

### 1. 参考图地址本身不可访问

这类问题看起来像模型失败，实际上常常是资源根本没读到。

### 2. 修改指令太宽

“帮我改得更好看”这类描述通常不够稳。  
尽量把你要保留什么、改变什么写清楚。

### 3. 同时要求改太多东西

第一次验证链路时，只改 1 到 2 个重点就够了。  
不要一口气要求换背景、换穿着、换构图、换光线。

### 4. 当前分组没开编辑模型

如果文生图可用、编辑图不可用，先查的不是 token 本身，而是这个模型是否在当前分组开放。

## 推荐动作

最稳的走法是：

1. 先拿一张稳定可访问的参考图
2. 只写一句很明确的修改要求
3. 先请求 `n = 1`
4. 成功后再逐步增加复杂度
