# 聊天

井陉当前提供两条主要聊天接口路径：

- `POST /v1/chat/completions`
- `POST /v1/messages`

对于大多数 SDK、客户端和后端服务，推荐优先使用 `chat/completions`。如当前工作流原生基于 Anthropic Messages 协议，可使用 `messages` 路径。

## 接口选择建议

### `/v1/chat/completions`

适用于以下场景：

- 通用 OpenAI 兼容接入
- 主流客户端与后端服务
- 首次 API 验证

### `/v1/messages`

适用于以下场景：

- 现有工作流基于 Anthropic Claude Messages 协议
- 需要保持 Claude 原生请求结构
- 已明确使用 `anthropic-version` 请求头

## 最小请求：Chat Completions

```bash
curl https://api.jingxing.uk/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "gpt-5.4",
    "messages": [
      {
        "role": "user",
        "content": "请用一句话介绍井陉。"
      }
    ]
  }'
```

该请求主要用于验证：

- 令牌是否有效
- Base URL 是否正确
- 当前模型是否可用
- 聊天主链路是否已建立

## 最小请求：Claude Messages

```bash
curl https://api.jingxing.uk/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 512,
    "messages": [
      {
        "role": "user",
        "content": "请用三句话总结这段内容。"
      }
    ]
  }'
```

使用该接口时，应特别注意以下字段：

- `anthropic-version`
- `max_tokens`

## 参数使用建议

`/v1/chat/completions` 当前已支持多项扩展能力，例如：

- `stream`
- `tools`
- `tool_choice`
- `response_format`
- `reasoning_effort`
- `modalities`
- `audio`

首次接入建议采用分阶段验证方式：

1. 先仅使用 `model + messages`
2. 再验证 `stream`
3. 再验证工具调用
4. 最后扩展多模态或音频能力

## 常见问题

### 模型不可见

常见原因是当前令牌或分组未开放目标模型。建议先执行 `GET /v1/models` 确认模型可见性。

### `messages` 结构错误

不同客户端导出的消息结构可能存在差异。出现 `400` 时，应优先校验消息数组结构。

### 未携带 `anthropic-version`

使用 `/v1/messages` 时，如未携带 `anthropic-version`，请求可能无法按预期工作。

### 首次验证即叠加过多参数

首次请求同时叠加流式、工具调用和长上下文，会增加排查复杂度。建议采用最小请求逐步扩展。

### 聊天可用但其他能力不可用

聊天能力验证通过并不自动代表图片、视频或音频能力已经开放，这些能力仍取决于当前分组与模型权限。

## 使用建议

建议按以下顺序进行聊天能力验证：

1. 执行 `GET /v1/models`
2. 选择一个常规聊天模型
3. 发送非流式最小请求
4. 成功后再接入客户端或扩展参数
