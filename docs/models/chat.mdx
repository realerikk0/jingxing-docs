# 聊天

这一页解决的是：

**我现在要让模型开始正常对话，应该先走哪条接口，最小请求怎么写。**

当前项目里已经明确暴露了两条聊天主链路：

- `POST /v1/chat/completions`
- `POST /v1/messages`

如果你只是想先把主流客户端、后端服务、脚本调通，**优先用 `/v1/chat/completions`**。  
如果你本来就是按 Anthropic Claude Messages 格式在发请求，再看 `/v1/messages`。

## 先说结论

- 通用接入优先走 `chat/completions`
- Claude 原生工作流再考虑 `messages`
- 第一次验证不要先上流式、工具调用、图像输入
- 先把一条最小对话跑通，再慢慢加复杂参数

## 最小请求：Chat Completions

```bash
curl https://api.jingxing.uk/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{
    "model": "gpt-5.4",
    "messages": [
      {
        "role": "user",
        "content": "请用一句话介绍井陉。"
      }
    ]
  }'
```

这是当前最稳的起点。  
只要这条能通，就说明你的：

- token
- Base URL
- 模型权限
- 基本聊天链路

至少已经是正常的。

## 什么时候要改用 `/v1/messages`

`/v1/messages` 更适合这些情况：

- 你原本就按 Anthropic SDK 或 Claude 原生协议在发请求
- 你需要保持 Claude Messages 的请求结构
- 你明确知道自己要带 `anthropic-version`

最小请求示例：

```bash
curl https://api.jingxing.uk/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 512,
    "messages": [
      {
        "role": "user",
        "content": "请用三句话总结这段对话的重点。"
      }
    ]
  }'
```

这里最容易漏掉的就是：

- `anthropic-version`
- `max_tokens`

## 先别急着动的参数

`/v1/chat/completions` 当前 OpenAPI 里已经列出了很多能力：

- `stream`
- `tools`
- `tool_choice`
- `response_format`
- `reasoning_effort`
- `modalities`
- `audio`

但如果你是第一次接井陉，建议先别一起上。  
最稳的顺序是：

1. 先只传 `model + messages`
2. 再试 `stream`
3. 再试工具调用
4. 最后再碰多模态或音频输出

## 最常见的 5 个问题

### 1. 模型名看起来对，但当前 token 根本不可见

这时候最容易被误判成“接口不兼容”。  
先跑 `GET /v1/models`，别靠记忆。

### 2. `messages` 格式写错

尤其是不同客户端导出的结构不一样时，最容易在这里出 `400`。

### 3. Claude 请求忘了带 `anthropic-version`

如果你走的是 `/v1/messages`，这一步一定要检查。

### 4. 一开始就开流式 + 工具 + 超长上下文

这样一旦报错，很难一眼知道是哪一层出问题。

### 5. 聊天接口能用，就以为所有能力都能用

聊天可用不代表：

- 图片一定可用
- 视频一定可用
- 音频一定可用

这些都还要看分组和模型开放情况。

## 推荐动作

如果你现在就是要验证聊天能力：

1. 先跑 `GET /v1/models`
2. 选一个最常规的聊天模型
3. 先发一条非流式最小请求
4. 成功后再接客户端或继续扩参数
