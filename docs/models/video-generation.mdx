# OpenAI 兼容视频生成

井陉当前提供 OpenAI 风格的视频任务接口，用于提交视频生成任务、查询执行状态并在任务完成后获取视频内容。本文用于说明该接口的使用场景、最小请求与常见问题。

## 可用路径

当前仓库已明确暴露以下路径：

- `POST /v1/videos`
- `GET /v1/videos/{task_id}`
- `GET /v1/videos/{task_id}/content`

## 适用场景

该路径通常适用于以下场景：

- 后端服务按 OpenAI 风格封装视频调用
- 需要统一使用 `/v1` 路径体系
- 需要将任务提交、状态轮询与内容下载分离处理

## 工作方式说明

该接口采用任务制流程，而非同步返回完整视频内容。典型流程如下：

1. 提交视频生成任务
2. 获取任务 `id`
3. 轮询查询状态
4. 任务完成后获取视频内容

## 最小请求

`/v1/videos` 当前使用 `multipart/form-data` 作为请求体格式。

```bash
curl https://api.jingxing.uk/v1/videos \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -F "model=sora-2" \
  -F "prompt=一只柯基在海边追逐浪花，暖色日落，镜头平稳推进" \
  -F "seconds=8"
```

如需使用参考图，可增加：

```bash
-F "input_reference=@/absolute/path/to/reference.png"
```

## 返回结果说明

任务创建成功后，通常会返回如下结构：

```json
{
  "id": "sora-2-123456",
  "object": "video",
  "model": "sora-2",
  "status": "queued",
  "progress": 0,
  "created_at": 1764347090922,
  "seconds": "8"
}
```

建议优先保存以下字段：

- `id`
- `status`

## 参数说明

### `model`

用于指定视频模型。建议在调用前确认当前分组开放的模型范围。

### `prompt`

首次验证建议使用较短、动作单一的提示词，以便快速判断链路是否正常。

### `seconds`

用于指定视频时长。首次验证建议使用较短时长。

### `input_reference`

用于提供参考图像。该字段为可选项，建议在纯文本视频任务验证完成后再引入。

## 与 Kling 路径的区别

`/v1/videos` 更适合需要维持 OpenAI 风格调用结构的场景。若目标是直接使用 Kling 风格接口，可参考：

- [Kling 文生视频](/docs/models/kling-text-to-video)
- [Kling 图生视频](/docs/models/kling-image-to-video)

## 常见问题

### 将任务接口误当作同步下载接口

该接口首先返回任务状态，而不是完整视频文件。

### 未填写 `seconds`

缺少时长字段是视频请求失败的常见原因之一。

### 聊天模型可用，但视频模型不可用

聊天能力可用并不表示视频能力已开放。应优先检查当前分组和模型权限。

### 首次验证即使用参考图

参考图会增加请求复杂度。建议先以纯文本任务完成首轮验证。

## 使用建议

建议按以下顺序验证视频链路：

1. 提交纯文本短视频任务
2. 保存返回的任务 `id`
3. 查询一次任务状态
4. 在任务完成后获取视频内容
